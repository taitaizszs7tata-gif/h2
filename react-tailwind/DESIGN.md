# H2 Architecture AI Project Design Rules

本文件用于约束当前 `react-tailwind` 项目的后续开发。它不是单纯说明文档，而是 AI 和开发者修改页面时必须遵守的项目规则。

## 一句话原则

保持当前产品的白色工作台风格、统一组件语言和完整页面跳转层级。不要为了实现局部需求而减少流程、跳过页面或创造第二套视觉系统。

## 固定页面流程

当前项目必须保留以下流程：

```text
登录
  -> 项目列表
  -> 选择项目
  -> 项目工作区
  -> AI 协同画布分类
  -> 公共/个人画布大厅
  -> 画布详情
```

### 流程禁止项

- 禁止登录成功后直达工作区或画布详情。
- 禁止项目列表点击后直达画布详情。
- 禁止顶部导航点击 `AI 协同画布` 后跳过分类页和大厅。
- 禁止删除公共/个人画布大厅。
- 禁止画布详情返回到项目列表；画布详情必须返回画布大厅。
- 禁止为单个功能临时新增一条绕开主流程的捷径。

## 当前路由和状态层级

| 层级 | 路由/状态 | 组件 | 说明 |
| --- | --- | --- | --- |
| 1 | `/` | `FloraSignIn` | 登录页 |
| 2 | `/projects` | `ProjectSelection` | 项目列表 |
| 3 | `view="workspace"` | `MainWorkspace` | 项目工作区 |
| 4 | `activeTab="canvas"` + `canvasSubview="categories"` | `CanvasCategories` | 画布分类 |
| 5 | `activeTab="canvas"` + `canvasSubview="lobby"` | `CanvasLobby` | 公共/个人画布大厅 |
| 6 | `activeTab="canvas"` + `canvasSubview="detail"` | `CanvasDetail` / `RemixCanvasDetail` | 画布详情 |

如果后续改成真实 URL，也必须保留同样层级。

## 视觉风格锁定

当前产品视觉基调：

- 主背景：白色。
- 主文字：黑色。
- 次级文字：中性灰。
- 面板：白色或极浅灰。
- 控件：浅灰底、浅灰边框。
- 主操作按钮：黑底白字。
- 强状态：绿色表示在线/已联通，玫红表示通知/协作提示。
- 风格：工作台、产品后台、可扫描、克制，不做营销页式大 hero。

## 颜色规则

业务页面不要继续散落新的 raw hex。已有旧代码可以保留，但新增代码优先使用统一 token 或既有组件。

当前视觉 token 来源：

- 项目内：`app/globals.css`
- 可复用设计系统：`../archboard-design-system/tokens/globals.css`

新增颜色前必须先问：

1. 现有 token 能不能表达？
2. 是否会破坏白色工作台风格？
3. 是否需要同步更新 `../archboard-design-system/DESIGN.md`？

## 组件规则

优先使用：

- `components/ui/*`
- `../archboard-design-system/components/ui/*` 的组件设计规则

新增或修改 UI 时：

- 按钮必须保持统一高度、圆角、hover、focus。
- 卡片必须保持统一比例、边框、圆角、图片填充方式。
- 输入框必须保持统一高度、边框、placeholder、focus。
- 面板必须保持统一边框、背景、内边距。
- 顶部导航必须保持高度和信息层级。
- 图标优先使用 `lucide-react`。

禁止：

- 在业务组件里复制一整段新的按钮样式。
- 新增第二套卡片视觉。
- 新增圆角过大的装饰卡片。
- 新增和当前页面风格不一致的渐变、装饰背景、视觉噪音。

## 画布详情规则

画布详情页目前由 `RemixCanvasDetail` 负责嵌入外部画布体验。修改时必须保证：

- 顶部全局导航仍保留。
- 详情区域占满顶部导航下方剩余高度。
- 返回按钮回到画布大厅。
- 不改变登录、项目列表、画布分类、画布大厅的前置流程。
- 外部画布可以更新，但不能接管整个应用路由层级。

## 登录规则

登录是系统入口，不能弱化成自动跳转。

必须保留：

- 登录成功进入 `/projects`。
- 登录失败提示统一错误，不指明账号或密码哪个错。
- 首次登录提示绑定手机号或邮箱。
- 忘记密码走绑定手机号/邮箱验证。
- 未绑定联系方式时提示联系管理员。
- 连续失败 5 次锁定 10 分钟。
- 网络错误和服务器错误使用不同提示。

## 项目列表规则

项目列表是工作区入口，不能被跳过。

必须保留：

- 工作区身份区域。
- 搜索项目能力。
- 项目卡片列表。
- 只有选择项目后才能进入工作区。

## 画布大厅规则

公共/个人画布大厅是进入详情前的必要层。

必须保留：

- 返回画布分类。
- 公共画布分区。
- 个人画布分区。
- 新建个人画布入口。
- 进入画布详情的明确点击行为。

## 修改前检查

每次修改前先确认：

- 这次修改是否会跳过任何中间页？
- 是否新增了和现有按钮/卡片/输入框/面板不一致的视觉？
- 是否改变了画布详情返回路径？
- 是否影响登录到项目列表的入口？
- 是否应该同步更新 `README.md` 或本文件？

## 修改后检查

完成后至少检查：

- 登录页仍可进入项目列表。
- 项目列表仍可选择项目。
- 工作区顶部导航仍存在。
- AI 协同画布仍先进入分类/大厅，再进入详情。
- 详情返回仍回到画布大厅。
- 视觉没有出现黑底黑字、白字不可见、空白画布丢内容等问题。

## 与通用设计系统的关系

通用设计系统在：


当前项目以本文件为本地最高约束；通用设计系统作为复用和新项目参考。后续如果当前项目沉淀出更稳定的规则，应同步回 `../archboard-design-system`。
