# User Flow Lock

## 主流程锁定

后续开发不得省略以下中间页：

```text
登录 -> 项目列表 -> 选择项目 -> 公共/个人画布大厅 -> 画布详情
```

项目实际实现中，在进入公共/个人画布大厅前还有 AI 协同画布分类页；如果新项目保留分类，则流程为：

```text
登录 -> 项目列表 -> 选择项目 -> AI 协同画布分类 -> 公共/个人画布大厅 -> 画布详情
```

## 禁止行为

- 禁止登录后直接进入画布详情。
- 禁止项目列表点击后直接进入画布详情。
- 禁止从顶部 `AI 协同画布` tab 直接打开详情页。
- 禁止绕过公共/个人画布大厅。
- 禁止把“新建个人画布”做成全局入口而跳过项目上下文。
- 禁止在忘记密码、登录失败、首次登录绑定提示中泄露具体错误项。

## 登录行为规则

- 登录成功：跳转到 `/projects`。
- 登录失败：提示“账号或密码错误”，不指明具体错误项。
- 首次登录：提示绑定手机号或邮箱。
- 已绑定手机号/邮箱：直接进入项目选择页。
- 忘记密码：通过已绑定手机号或邮箱验证后重置密码。
- 未绑定手机号/邮箱：提示“请联系管理员重置密码”。
- 连续失败 5 次：锁定 10 分钟，提示剩余时间。
- 网络错误：提示“网络异常，请重试”。
- 服务器错误：提示“服务暂时不可用”。

## 画布行为规则

- 画布详情顶部必须保留项目全局导航。
- 详情页的返回按钮必须回到画布大厅。
- 公共画布和个人画布要在大厅中明确分区。
- 个人画布可以发送到公共画布，但不能自动覆盖公共画布。
- 外部嵌入画布时，外层应用仍负责主流程和导航层级。
